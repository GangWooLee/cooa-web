# 비전 기반 디자인 리뷰 파이프라인 (2026-07-09~10 실증)

전 화면 UI/UX 기본기 리뷰를 코드-only가 아니라 **스크린샷 비전 기반**으로 수행해
6일치 백로그(53 findings + 실측 6건)를 이틀에 리뷰→적용까지 완주한 레시피. 재사용 가치가 높다.

## 파이프라인 (5단)

1. **성문화 먼저**: 리뷰 전에 기존 토큰·헬퍼 관행을 `DESIGN.md`로 성문화. 리뷰 기준이 "취향"이
   아니라 "문서 위반"이 된다 (Specs beat vibes). 미결정은 §에 "확정 대기"로 명시하고 결정 후 기입.
2. **기준선 캡처**: 시드 계정 피커(비밀번호 없음) 덕에 Playwright 스크립트로 전 화면
   × 3뷰포트(1440/768/390) × 역할별 캡처가 자동화된다. 캡처 후 반드시 사람이(모델이) 이미지를
   직접 통독 — 코드 grep이 못 잡는 문제(여백 리듬·정보 밀도·fit 미흡)는 눈이 잡는다.
3. **다차원 파인더 + adversarial 검증**: 차원별(IA·레이아웃·타이포·스페이싱·색·폼·상태·일관성·
   반응형·a11y) 파인더가 스크린샷+코드를 함께 보고 file:line까지 특정 → 회의적 검증자가 스크린샷·
   코드 재확인으로 기각/조정. 이번 실측: 55건 중 2건 기각·10건 심각도 조정 — 검증 패스가 가짜 양성을
   실제로 걸렀다.
4. **WP 묶음 적용**: findings를 파일 겹침 기준으로 묶어(WP) 순차 적용. 겹치는 파일이 많아
   병렬 워커는 금물 — 같은 브랜치 원자 커밋 열차가 정답.
5. **after 재캡처 → 비전 대조**: 같은 스크립트를 OUT만 바꿔 재실행, 전/후를 눈으로 대조.

## 함정 (실측)

- **`rails server -d`(데몬화)는 Puma에서 조용히 실패한다** — "Booting Puma"까지 찍고 리스닝 없음.
  백그라운드 셸 + pidfile + readiness 루프(curl 200까지)로 띄워라. 포트는 사용자 dev 서버(3000)와
  충돌 피해 3100.
- **개발 DB를 owner로 세면 RLS에 속는다**: 기본 연결(cooa_app)로 `Account.count`=0이어도 데이터가
  있을 수 있다(R8의 역방향). 시드 여부 판단은 owner 연결로.
- **스크린샷의 Bullet Warnings 오버레이**(개발 전용 N+1 게이트)는 리뷰에서 무시 대상으로 브리프에
  명시해야 파인더가 헛다리를 안 짚는다.
- **commit-msg 훅은 "트레일러 전무"를 사람 커밋으로 통과시킨다** — 워커가 트레일러를 빼먹으면
  차단이 아니라 조용히 통과. 워커 브리프에 트레일러 두 줄을 복붙으로 박아라.
- **성공 상태 ≠ 빈 상태**: 스크리닝 all-clear("위반·주의 사항 없음")를 중립 빈 상태로 강등하면
  a11y·시맨틱 회귀다. 빈 상태 표준화 때 성공 표면은 예외로 지켜라.
- **색 대비는 토큰 교체가 아니라 신호 재배치로 푼다**: 앰버/그린은 이 팔레트에서 소형 텍스트 AA가
  구조적으로 불가 → 라벨을 ink로, 색은 아이콘·보더로 옮기는 3중 신호 재구성이 정답이었다.
  seq 뱃지도 "흰 글자+색 채움"→"아웃라인+ink"로 바꾸면 어떤 색이 와도 대비 문제가 소멸한다.
